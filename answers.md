1. Чому важливо вказувати user.name та user.email? Це необхідно для ідентифікації автора змін. Git додає цю інформацію до кожного коміту. Це дозволяє відстежувати історію розробки: хто, що і коли змінив. Крім того, сервіси на зразок GitHub використовують email для того, щоб пов'язати коміти з вашим профілем (аватаркою та статистикою).

2. Яка різниця між git pull і git fetch?

git fetch: Завантажує зміни з віддаленого репозиторію (GitHub) у вашу локальну папку .git, але не змінює ваші робочі файли. Це безпечний спосіб подивитися, що нового, не ламаючи свою роботу.

git pull: Це комбінація двох команд (git fetch + git merge). Вона завантажує зміни і одразу намагається об'єднати (злити) їх з вашою поточною гілкою.

3. Як скасувати останній коміт, не втрачаючи зміни у файлах? Для цього використовується команда git reset з прапором --soft. Команда: git reset --soft HEAD~1 Ця команда скасовує факт створення коміту, але залишає всі ваші зміни у файлах (в області Staging), дозволяючи вам внести правки і зробити коміт зно
